# Create Issue

GitHub issueをgh CLIで作成するスキル。

## 前提条件

- `gh` CLIがインストール・認証済みであること
- gitリポジトリ内で実行すること

## ユーザーへの質問方針

**必ずAskUserQuestionツールの選択式UIを使って情報を収集する。** テキスト入力を求めない。

- 選択肢から選ぶだけで回答できるようにする
- ユーザーが自由記述したい場合は「Other」オプションで対応できる
- 一度に最大4問まで、関連する質問をまとめて聞く

## ワークフロー

### 1. リポジトリ情報の確認

```bash
gh repo view --json name,owner,hasIssuesEnabled
```

issueが無効なリポジトリの場合はユーザーに通知して終了する。

### 2. issueテンプレートの検出

以下のパスを確認してテンプレートの有無を判定する：

- `.github/ISSUE_TEMPLATE/` ディレクトリ
- `.github/ISSUE_TEMPLATE.md`

テンプレートが見つかった場合は中身を読み、対応するテンプレートを使用する。

### 3. issue内容の収集（選択式）

AskUserQuestionで選択式の質問を使って情報を集める。

**第1回の質問（まとめて聞く）：**

```
質問1: issueの種類は？
  header: "種類"
  options:
    - label: "バグ報告"
    - label: "機能リクエスト"
    - label: "改善提案"
    - label: "質問"

質問2: 優先度は？
  header: "優先度"
  options:
    - label: "高"
    - label: "中"
    - label: "低"

質問3: 担当者をアサインする？
  header: "アサイン"
  options:
    - label: "自分にアサイン"
    - label: "アサインなし"
```

**第2回の質問（種類に応じて）：**

バグ報告の場合：
```
質問1: 重大度は？
  header: "重大度"
  options:
    - label: "クラッシュ/データ損失"
    - label: "機能が使えない"
    - label: "回避策あり"
    - label: "軽微な不具合"

質問2: 再現性は？
  header: "再現性"
  options:
    - label: "常に再現する"
    - label: "時々再現する"
    - label: "まれに再現する"
    - label: "未確認"
```

機能リクエストの場合：
```
質問1: 機能の規模は？
  header: "規模"
  options:
    - label: "小さな改善"
    - label: "新機能追加"
    - label: "大規模な変更"

質問2: 緊急性は？
  header: "緊急性"
  options:
    - label: "すぐに必要"
    - label: "次のリリースまでに"
    - label: "いつかあれば嬉しい"
```

ユーザーの最初のメッセージから文脈が十分に読み取れる場合は、既に分かっている情報の質問はスキップする。

### 4. issue本文の自動生成

収集した選択肢の回答と、ユーザーの最初のメッセージの内容を基に、issue本文を自動生成する。

#### バグ報告の場合

```markdown
## 概要
[ユーザーのメッセージから抽出]

## 再現手順
[ユーザーのメッセージから推測・構成]

## 期待される動作
[文脈から推測]

## 実際の動作
[ユーザーのメッセージから抽出]

## 環境
[分かる範囲で記載]

## メタ情報
- 重大度: [選択結果]
- 再現性: [選択結果]
```

#### 機能リクエストの場合

```markdown
## 概要
[ユーザーのメッセージから抽出]

## モチベーション
[文脈から推測]

## 提案する解決策
[ユーザーのメッセージから抽出]
```

### 5. 最終確認（選択式）

生成したタイトルと本文をユーザーに表示し、AskUserQuestionで確認する：

```
質問: この内容でissueを作成しますか？
  header: "確認"
  options:
    - label: "作成する"
    - label: "タイトルを修正"
    - label: "本文を修正"
    - label: "キャンセル"
```

「タイトルを修正」「本文を修正」が選ばれた場合は修正案を選択式で提示するか、Otherで自由入力を受け付ける。

### 6. issueの作成

```bash
gh issue create --title "タイトル" --body "本文"
```

必要に応じてラベルやアサインも付与する：

```bash
gh issue create --title "タイトル" --body "本文" --label "bug" --assignee "@me"
```

### 7. 結果の報告

作成されたissueのURLをユーザーに報告する。

## 注意事項

- テンプレートがある場合はテンプレートの形式に従う
- ラベルの付与はリポジトリに存在するラベルのみ使用する（`gh label list`で確認）
- 日本語・英語はリポジトリの既存issueの言語に合わせる
